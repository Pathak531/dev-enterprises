## ğŸ“¦ Available Scripts

Inside the project directory, the following commands are available:

### **1. `npm start`**

Starts the application in **development mode**.

* Opens the app automatically at **[http://localhost:3000](http://localhost:3000)**.
* Hot reloads the UI on code changes.
* Displays lint warnings and errors in the console.

### **2. `npm test`**

Runs the test suite in **interactive watch mode**.

* Automatically re-runs tests on file updates.
* Supports Jest-based unit testing provided by CRA.

### **3. `npm run build`**

Creates an optimized **production-ready** build.

* Outputs files to the `build/` folder.
* Minifies JavaScript and generates hashed filenames.
* Ensures best performance for deployment.

### **4. `npm run eject`** ğŸ”¥

**Warning:** Ejecting is irreversible.

* Exposes configuration files such as Webpack, Babel, ESLint, and more.
* Recommended only when full control over the build system is required.
* All CRA-managed scripts become local configuration files.
